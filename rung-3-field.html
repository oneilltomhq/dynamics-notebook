01 · the landscape of dice

The landscape of dice

This is noise(x, y, t). Not dice rolled per frame — a smooth function you sample. Same input, same output, every time. Nearby inputs, similar outputs. Bright is high, dark is low, and the whole landscape drifts slowly through time.

n = noise(x * scale, y * scale, t) // same inputs → same answer, always push = σ * (n * 2 - 1) // centered — smooth in x, y, and t
/field/scale feature px90
/field/speed0.35
/field/octaves3

This is a function, not a texture. Ask it the same (x, y, t) twice and it answers the same number twice — deterministic randomness. Under the hood: a hash pins one fixed random value to every integer lattice corner, and smooth interpolation fills the gaps. The octaves slider is the other trick: each octave doubles the frequency and halves the volume, then they sum. Octaves stack detail — big swells + small ripples = fbm. At 1 octave it's soap bubbles; at 5 it starts to look like weather.

02 · the cliffhanger, resolved

The cliffhanger, resolved

The exact crowd from rung 2 — forty spring-tethered dots, k = 45, c = 5, nothing changed. Except the random force no longer comes from dice. Each dot samples the field at its own home position, and the field scrolls past. One slider sweeps the feature size.

/wind/scale feature px180
/wind/strength σ1800

This one slider interpolates between rung 2's two failure modes; wind lives in the middle. Drag it to the right: features grow past the size of the whole stage, every dot reads the same value — rung 2's shared dice, the rigid sheet. Drag it to the left: features shrink below the dot spacing, neighbors decorrelate — rung 2's independent dice, boiling static. Park it in the middle and watch: gusts travel across the crowd, neighbors lean together, distance disagrees. That's wind. Nothing was added — only where the randomness lives changed, from time to space.

03 · grass

Grass

Sixty blades, base pinned. Each blade leans toward whatever the field says at its base — and each segment chases the segment below it with a rung-1 damped spring, so the tip lags the root. Wind field, spring chain, nothing else.

/grass/stiffness90
/wind/strength130

The haze behind the blades is the same field they're sampling — watch a bright gust drift through and the blades under it bow in sequence. This is the rung-2 promise paying off: the texture is entirely in the parameters. Stiff + weak wind reads as wheat stubble; loose + strong reads as underwater kelp. Same twelve lines of code.

04 · weather for parameters

Sample it for anything

Rung 1's move was "spring anything" — any number can be a spring target. Same move here: any number can sample the field. This ring's swell, glow, and lift each read the same field at three different fixed points, marked below.

/field/speed0.50

Three probes, one field: each parameter wanders smoothly and independently, yet when a broad bright patch drifts over all three, the whole ring gets wilder together — coherence for free. Rung 2 gave parameters a drunk walk; rung 3 gives them weather. One cliff remains: the obvious next move — pushing particles along the field's slope — makes them pile up in noise clumps and stall. Rung 4 fixes that with one rotation.
